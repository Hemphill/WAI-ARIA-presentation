WAI

The Web Accessibility Initiative of the W3C

Strategies, guidelines, & resources to make the Web accessible to people with disabilities

WAI Develops

  • guidelines widely regarded as the international standard for Web accessibility
  • support materials to help understand and implement Web accessibility
  • resources, through international collaboration

WAI Guidelines

  • Authoring Tool Accessibility Guidelines (ATAG)
  • User Agent Accessibility Guidelines (UAAG)
  • Independent User Interface (Indie UI) - closed
  • Evaluation and Report Language (EARL)
  • Web Content Accessibility Guidelines (WCAG)
  • Accessible Rich Internet Applications (WAI-ARIA)

What's the difference?

WCAG

Tends to answer the question “What should we be doing?”

ARIA

Tends to answer the question “How do we accomplish it?”

WAI-ARIA

Accessible Rich Internet Applications

Navigation & Context

ARIA is a gap-filler for HTML semantics. It is used when semantic elements in HTML fall short of providing contextual meaning.

Semantic 1st

It is preferred to find a semantic way to use HTML over ARIA elements, but ARIA is there when HTML falls short.

ARIA Provides

  • Roles
    for HTML elements through the role attribute
  • States & Properties
    through aria- prefixed attributes

Roles

An ARIA pre-determined list of roles

role attribute makes the element's context obvious to assistive technologies

Increases navigation, and other functionality for each role

Example: Roles in a Bootstrap 3 Grid

<div role="grid">
    <div class="row" role="row">
        <div class="col-md-8" role="gridcell">
          content here...
        </div>
        <div class="col-md-4" role="gridcell">
          content here...
        </div>
    </div>
</div>

Example: Roles in a Bootstrap 3 Alert

<div class="alert alert-success" role="alert">
  message goes here...
</div>

Landmark Roles

Aid assistive technology in navigating the page.

  • application
  • banner
  • complementary
  • contentinfo

  • search

Landmark Role

application

role="application"

“A region declared as a web application, as opposed to a web document.”

Be aware

The main function of the application role is to allow your app to use custom shortcut keys.

Use it sparingly

If your page is only using the brower's default shorcuts, then avoid using the application role.

Landmark Role

banner

role="banner"

“A region that contains mostly site-oriented content, rather than page-specific content.”

Think “header”

“Site-oriented content typically includes things such as the logo or identity of the site sponsor, and site-specific search tool. A banner usually appears at the top of the page and typically spans the full width.”

Example: Banner Landmark Role

<header role="banner">
  <img src="images/logo.png" alt="Webstop Logo">
  <p>The Grocery Website Specialists</p>
</header>

Landmark Role

complementary

role="complementary"

“A supporting section of the document, designed to be complementary to the main content at a similar level in the DOM hierarchy, but remains meaningful when separated from the main content.”

Example: Complementary Landmark Role

<main role="main" class="weekly-ad">
  <h1>Weekly Ad</h1>
  <article>Ad item 1</article>
  <article>Ad item 1</article>
</main>
<aside role="complementary" class="shopping_list">
  <h1>Weekly Ad</h1>
  <ul>
    <li>Shopping list item 1</li>
    <li>Shopping list item 2</li>
  <ul>
</aside>

Landmark Role

contentinfo

role="contentinfo"

“A large perceivable region that contains information about the parent document.”

Think “footer”

“Examples of information included in this region of the page are copyrights and links to privacy statements.”

Example: ContentInfo Landmark Role

<footer role="contentinfo">
  <p>© 2016 Webstop.com, Inc. All Rights Reserved</p>
  <p><a href="docs/privacy">Privacy Policy</a></p>
</footer>

Landmark Role

form

role="form"
<form>

“A landmark region that contains a collection of items and objects that, as a whole, combine to create a form.”

Just use HTML <form> tags

Landmark Role

main

role="main"
<main>

“The main content of a document.”

The page “body”

“This marks the content that is directly related to or expands upon the central topic of the document.”

RIP “skip to main content” technique

“The main role is a non-obtrusive alternative for "skip to main content" links, where the navigation option to go to the main content.”

Example: Main Landmark Role

<main role="main" class="weekly-ad">
  <h1>Weekly Ad</h1>
  <article>Ad item 1</article>
  <article>Ad item 1</article>
</main>

Landmark Role

navigation

role="navigation"
<nav>

“A collection of navigational elements (usually links) for navigating the document or related documents.”

Example: Main Landmark Role

<nav role="navigation" class="site-nav">
  <ul>
    <li><a href="/index">Home</a></li>
    <li><a href="/Recipes">Recipes</a></li>
  </ul>
</nav>

Landmark Role

search

role="search"

“A landmark region that contains a collection of items and objects that, as a whole, combine to create a search facility.”

Example: Search Landmark Role

<form role="search">
  <label for="site-search">Site Search</label>
  <input type="search" name="site-search">
  <button type="submit">Search</button>
</form>

Example: All Landmark Roles

<!doctype html>
<html>
  <head>
    <title>Title of your page</title>
    <meta charset="utf-8">
    <meta name="description" content="description of your page">
  </head>
  
  <body>

    <header role="banner">
      <h4>Title of your web site</h4>
      <nav role="navigation">
        <ul>
          <li><a href="/index">Home</a></li>
          <li><a href="/recipes">Recipes</a></li>
          <li><a href="/contact">Contact Us</a></li>
        </ul>
        <form role="search">
          <input type="search" placeholder="Site Search">
          <button type="submit">Search</button>
        </form>
      </nav>
    </header>

    <main role="main">
      <h1>Main content & page title</h1>
      <p>Awesome content here</p>
    </main>

    <aside role="complementary">
      <h2>Sidebar</h2>
      <p>Complementary content</p>
    </aside>

    <footer role="contentinfo">
      <h2>Footer</h2>
      <p>Information about the site and page</p>
      <p>© 2016 Webstop All Rights Reserved</p>
      <p><a href="/privacy">Privacy Policy</a></p>
    </footer>

  </body>
</html>
